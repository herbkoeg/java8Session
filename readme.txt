Lamdas

- FunctionalInterface: immer genau eine nicht implementiert Methode
- Lamdas sind lazy, d.h. werden werden nicht bei der Definition ausgeführt


Allgemein

- Default  implementierung in Interface
- Optionals in Pojos aufnehmen


Implementierungen von Functional-Interfaces:

Im Prinzip hat sich herausgestellt, dass bei der Programmierung man sich auf 4 typische Aktionen begrenzen kann

- Supplier: erzeuge was !
- Predicate: überprüfe bzw. teste -> true, false !
- Function: transformiere von einem Datentyp zu einem anderen Datentyp
  UnaryOperator: transformiere von einem Datentpy zum gleichen Datentyp
- Consumer: Nimm Datentyp entgegen, z.B. print auf sysout, foreach benötigt Consumer
